# Contributing to SourceHub

Thank you for considering contributing to SourceHub!

## How to Contribute

### Reporting Issues

Before creating an issue:
- Search existing issues to avoid duplicates
- Use the issue template if available
- Include reproduction steps and expected behavior

### Pull Requests

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Write/update tests if applicable
5. Commit with descriptive messages
6. Push to your fork and submit a PR

### Commit Messages

Use conventional commit format:
- `feat: add new feature` - new features
- `fix: resolve issue` - bug fixes
- `docs: update documentation` - docs changes
- `refactor: restructure code` - code refactoring
- `test: add tests` - adding tests

### Branches

| Branch | Purpose |
|--------|---------|
| `main` | Stable releases |
| `feature/*` | New features |
| `fix/*` | Bug fixes |

## Code Style

- Follow existing code patterns in the repository
- Run linting before submitting
- Write tests for new functionality
